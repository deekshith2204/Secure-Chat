Security model

  • Private key stays only in this browser.
  • Server stores only public keys, OTPs, and ciphertext.
  • Messages are encrypted client-side using Web Crypto API.

Project demo notes

Default API base is http://localhost:8000. Update API_BASE in app.js for Azure deployment.

  1. Open the app in two browsers or profiles.
  2. Register two users with different emails.
  3. Generate keys in each browser.
  4. Send encrypted messages between them.
  5. Fetch messages and observe local decryption only in the receiver browser.
